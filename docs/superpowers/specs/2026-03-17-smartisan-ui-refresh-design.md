# Smartisan 风格全站 UI 改造设计文档

- 日期：2026-03-17
- 项目：wallpapers（frontend）
- 范围：全站（首页、管理页、登录页、关于页、全局导航与基础控件）
- 设计决策：
  - 主风格：A「极简白灰」
  - 参考强度：2「中等还原（保留现有信息结构，替换视觉语言）」

## 1. 背景与目标

当前站点使用高饱和玻璃态（强渐变、重光效、较多浮动感），与 Smartisan 官网主流视觉语言不一致。

本次目标是在不改变核心业务流程与信息架构前提下，完成全站视觉系统升级：

1. 统一为白灰极简体系，降低视觉噪声。
2. 形成一致的导航、卡片、按钮、输入、弹窗风格。
3. 保持现有页面结构与交互路径，避免功能回归。

## 2. 非目标（Out of Scope）

1. 不新增业务功能，不改动 API 协议。
2. 不重构路由结构，不调整页面核心信息架构。
3. 不引入新的 UI 组件库，仅在现有 Vue + TDesign 基础上做样式覆盖。

## 3. 全局视觉系统

### 3.1 颜色与层级

- 页面背景：`#f5f5f7`
- 主卡片背景：`#ffffff`
- 主文字：`#111111`
- 次文字：`#666666`
- 边框：`#e5e5e5`
- 分割线：`#ececec`
- 主按钮深色：`#222222`（hover 轻微提亮）
- 危险态：红色描边 + 浅红背景（低饱和）

规则：强调色只用于“主操作/当前选中”，禁止大面积高饱和渐变。

### 3.2 形态与质感

- 圆角：统一 `12-16px`，导航胶囊可用更大圆角。
- 边框：以 `1px` 细边框为主。
- 阴影：轻阴影（短半径、低透明度），禁止重投影和强发光。
- 背景：去除玻璃拟态、去除大面积彩色光斑。

### 3.3 字体与排版

- 字体栈：`PingFang SC`, `-apple-system`, `BlinkMacSystemFont`, `Segoe UI`, `sans-serif`
- 标题与正文不再使用当前偏展示型字体，回归系统字体一致性。
- 文案层级通过字重与字号区分，不靠高饱和颜色区分。

### 3.4 动效

- hover 仅允许轻微阴影变化或 `1-2px` 位移。
- 过渡时长控制在 `150-220ms`。
- 保留 `prefers-reduced-motion` 降级策略。

## 4. 全局结构与组件规则

### 4.1 App 顶部导航（`App.vue`）

改为 Smartisan 倾向的双层结构：

1. 上层深色细条：承载品牌与状态信息。
2. 下层浅色主导航：承载主要路由与登录/退出动作。

约束：

- 主导航项使用浅灰底 + 深色字的选中逻辑。
- 移除现有玻璃态背景与蓝色高光态。

### 4.2 通用卡片（`.card` / `.page-container`）

- 全站统一白底卡片、细边框、轻阴影。
- 页面容器弱化装饰，突出内容模块。

### 4.3 表单控件与弹窗（TDesign 覆盖）

- 按钮：主按钮深色实底，次按钮白底描边。
- 输入框/选择框：白底、灰边框、聚焦时深灰边框 + 极轻焦点环。
- 弹窗：白底、细边框、圆角统一，弱化浮层滤镜。

## 5. 页面级改造

### 5.1 首页（`HomeView.vue`）

保留现有信息架构（Hero + 分类 + 搜索 + 图片网格 + 详情弹窗）：

1. Hero 区移除彩色渐变，改为白底信息块。
2. 分类按钮改为低饱和灰阶胶囊，选中态高对比但不鲜艳。
3. 图片卡片改为白底边框体系，保留图片展示；底部遮罩改为更轻的黑透明层。
4. 分类角标使用灰阶语义色（而非高饱和色块）。

### 5.2 管理页（`UploadView.vue`）

1. 上传区、图库区统一卡片语言。
2. 分类切换采用浅灰分段式按钮。
3. 删除按钮改为红色描边危险按钮，保留清晰风险提示。

### 5.3 登录页（`LoginView.vue`）

1. 采用中心单卡片布局。
2. 主按钮改为深色实底；移除现有蓝色渐变和重阴影。
3. 输入框与说明文本统一灰阶视觉。

### 5.4 关于页（`AboutView.vue`）

1. 保留三列内容结构。
2. 统一标题层级、段间距、行高。
3. 去除装饰性背景渐变，改为简洁信息卡。

## 6. 实施边界与兼容性

1. 仅修改前端样式与必要结构，不改业务逻辑函数签名。
2. 保留现有响应式断点，确保移动端可读可点。
3. 不引入额外依赖。

## 7. 验收标准

### 7.1 视觉一致性

1. 全站不再出现玻璃态背景、强蓝青渐变、重发光阴影。
2. 导航、卡片、按钮、输入框、弹窗风格一致。
3. 首页/管理/登录/关于在同一设计语言下无明显割裂。

### 7.2 交互与可用性

1. 分类筛选、搜索、上传、删除、下载、登录流程行为不变。
2. 可点击元素 hover/focus 状态明确。
3. 移动端无横向滚动，触控区域不低于 44px。

### 7.3 回归风险控制

1. 不影响现有 API 请求与数据渲染。
2. 不改变路由可达性和权限跳转逻辑。
3. 构建与类型检查通过。

## 8. 交付清单（预期）

1. `frontend/src/assets/styles/global.css`：重建全局 token 与通用组件视觉。
2. `frontend/src/assets/main.css`：调整页面背景与基础排版栈。
3. `frontend/src/App.vue`：双层导航与按钮样式改造。
4. `frontend/src/views/HomeView.vue`：首页模块视觉重塑。
5. `frontend/src/views/UploadView.vue`：管理页视觉重塑。
6. `frontend/src/views/LoginView.vue`：登录页视觉重塑。
7. `frontend/src/views/AboutView.vue`：关于页视觉重塑。

## 9. 风险与对策

1. 风险：TDesign 默认样式覆盖不完全导致局部割裂。
   - 对策：集中在全局样式中覆盖关键控件（button/input/select/dialog）并做页面抽检。
2. 风险：样式改造后局部断点布局溢出。
   - 对策：在 `375/768/1024/1440` 四档回归检查。
3. 风险：过度还原造成品牌辨识下降。
   - 对策：采用“中等还原”，保留当前产品文案与信息架构。
